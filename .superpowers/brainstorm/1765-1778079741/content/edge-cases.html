错误处理与边界情况

各种异常场景的行为定义

场景 行为
交易记录不存在 API 返回 code: -1，"transaction not found"，前端 toast 提示
关联时日期为空 前端校验：未选择日期不允许确认，按钮置灰
该日期无 KeyEvent 自动新建空 KeyEvent（date 已设置，title/content/color 均为空），然后关联
重复关联同一交易到同一日期 直接覆盖，视为更新（幂等操作）
KeyEvent 被删除后 关联交易记录的 key_event_date 保留旧值。查询关联交易时，KeyEvent 不存在则返回空列表。数据不脏 — 下次关联时会重建 KeyEvent。
解除关联后该日期无其他关联 不删除 KeyEvent（可能用户写了内容），保留事件记录
关联交易 Tab 数据为空 显示空状态提示："暂无关联交易记录"
跨账本关联 不同账本的交易均可关联到同一天。关联交易 Tab 中显示所有账本的记录。每条记录需标明所属账本。

关联交易表格额外列

由于跨账本场景，关联交易表格需要多一列显示账本名称。同时省略"类型"和"标记"列以控制宽度：

账本 分类 标签 描述 金额 操作

6 列，宽度可控。金额列通过颜色区分收入/支出/转账。
