API 接口设计

三个新接口，遵循现有 POST/JSON 风格

1. POST /api/v1/tr/link — 关联交易到关键事件

// Request
{ "transaction_id": "uuid-xxx", "date": "2026-05-06" }

// Response（成功）
{ "code": 0, "msg": "", "data": "2026-05-06" }

// Response（交易记录不存在）
{ "code": -1, "msg": "transaction not found", "data": null }

若该日期 KeyEvent 不存在，service 层自动创建空记录（title/content/color 为空）。

2. POST /api/v1/tr/unlink — 解除关联

// Request
{ "transaction_id": "uuid-xxx" }

// Response
{ "code": 0, "msg": "", "data": "uuid-xxx" }

仅清空 key_event_date，不删除 KeyEvent。

3. GET /api/v1/tr/linked/:date — 查询某日期关联交易

// Request: GET /api/v1/tr/linked/2026-05-06

// Response
{
  "code": 0,
  "msg": "",
  "data": [
    {
      "transaction_id": "uuid-xxx",
      "ledger_id": "...",
      "price": 12800,
      "transaction_type": "expense",
      "category": "餐饮",
      "description": "午餐",
      "flags": "",
      "key_event_date": "2026-05-06",
      "transaction_at": 1746518400,
      "created_at": 1746518400,
      "updated_at": 1746518400
    }
  ]
}

返回完整的 TransactionRecord 列表，跨所有账本（因为关键事件是全局的）。
